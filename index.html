Mentor Mentee

Project Overview

Key Features

  • Search and display instructors and students
  • View detailed instructor-student group assignments
  • Theme switching (light, dark, automatic)
  • Pagination and dynamic data loading

System Architecture

Frontend

  • HTML
  • CSS
  • JavaScript
  • Frappe Framework
  1. Responsive UI with Bootstrap for layout
  2. Dynamic rendering of intsructor and student lists
  3. Theme switcher for user customization

Backend

  • Python (Frappe Framework)
  1. Handles data retrieval and processing
  2. Whitelisted API for secure data access

Database

  • MariaDB
  1. Frappe built-in database (MariaDB)

Key Functionalities

Search Functionality

  • Real-time search by name for instructors and students.
  • Filters results dynamically using AJAX calls.

Instructor Details

  • Displays assigned student groups and their members with class

Student Details

  • Shows assigned instructors and guardian information.
  • Tooltip with student image and guardian names.

Pagination

  • Load more data with configurable page lengths (20, 100, 500, 2500).

Theme Switching

  • Light, dark, and automatic modes based on system preferences.

    TODO GNDEC System

    Project Overview

    Purpose

    • Streamlines task assignment and tracking across roles (Principal, HOD, Instructor, Student) in an educational setting.

    Key Features

    • Role based task creation and assignments
    • Task status and response updates.(Pending, Completed)
    • Bulk actions (delete, update status)
    • Search, Filters, and sort tasks with pagination
    • Email notifications for task assignments

    Target Users

    1. Principal
    2. HODs
    3. Instructors
    4. Students

    System Architecture

    Frontend

    1. HTML
    2. CSS
    3. Javascript
    4. Frappe framework
    5. Bootstrap
    6. Jinja
    • Responsive UI with modals for task creation and editing.
    • Dynamic table for task listing with sorting and filtering.

    Backend

    1. Python (Frappe Framework)
    • Handles task CRUD operations and role-based logic.
    • Whitelisted APIs for secure data access.

    Database

    1. Frappe’s built-in database MariaDB

    Key Functionalites

    Role-Based Task Assignment:

    • Principal: Assign tasks to HODs or self.
    • HOD: Assign tasks to Instructors or self.
    • Instructor: Assign tasks to Students or self.
    • Student: Create personal tasks for self.

    Task Management:

    • Create, edit, delete tasks with description, status, due date, and priority.
    • Update task response (Pending, Completed) by assignees.

    Bulk Operations:

    • Bulk Delete/Update: Delete or update status of multiple tasks.

    Interactive Features:

    • Search tasks by description.
    • Filter by status, response, or assignee.
    • Sort by fields (e.g., creation, due date).

    Notifications:

    • Email Alerts: For new or reassigned tasks.

    Technical Highlights

    Frontend:

    • Jinja2 Templating: For dynamic rendering.
    • JavaScript: For interactivity (e.g., modals, checkboxes, AJAX calls).
    • Bootstrap: For responsive design and modals.

    Backend:

    • Python Functions: get_context, get_todo_list, bulk_update_todos for task management.
    • Frappe ORM: For database queries (e.g., fetching users, tasks).
    • Role-Based Access Control: For secure operations.

    Security:

    • CSRF Tokens: For form submissions.
    • Whitelisted APIs: @frappe.whitelist for authenticated access.
    • Validation: Of assignees based on roles and department.

    User Interface

    Layout:

    • Task Creation Form: With assignee selection modal.
    • Task Table: Displaying tasks with sortable columns.
    • Navigation Tools: Filter modal and search bar for task navigation.

    Interactivity:

    • Dropdowns: For task response updates.
    • Modals: For editing tasks and viewing full descriptions.
    • “New” Badge: For unseen tasks assigned to the user.

    Implementation Details

    Data Flow:

    • Task Submission Flow: User submits task → Frontend sends POST request → Backend validates and saves → Email sent to assignee.
    • Live Updates: Task list updates via AJAX calls to get_todo_list.

    Key Components:

    • todo.html: Renders UI and handles client-side logic.
    • todo.py: Manages server-side logic, role checks, and notifications.

    Dependencies:

    • Frappe Framework: For backend and frontend utilities.
    • Bootstrap and jQuery: For UI components.

    Challenges:

    • Concurrency: Handling concurrent task updates with retry logic.
    • Access Control: Ensuring role-based restrictions for task assignments.
    • Scalability: Managing large task lists with pagination.

    Benefits and Impact

    • Efficiency: Simplifies task delegation and tracking across hierarchies.
    • User Experience: Intuitive UI with role-specific views and notifications.
    • Accountability: Tracks task status and responses, enhancing responsibility.
    • Scalability: Supports multiple users with pagination and optimized queries.

    Conclusion

    Summary:

    The ToDo GNDEC System is a secure, role-based platform for efficient task management in education.

    Key Takeaways:

    • Seamless Task Management: Seamless task assignment and tracking.
    • Access & Notifications: Robust role-based access and notifications.
    • Scalability: Potential for scalability and enhancements.
